Insecure Direct Object Reference (IDOR) is a type of access control vulnerability that arises when an application uses user-supplied input to access objects directly.

Imagine a website where you can view your profile at /profile/123. An IDOR vulnerability would exist if you could change the ID to /profile/124 and see another user's profile without being authorized.

How Attackers Exploit IDOR

  • URL Tampering: The attacker modifies parameters in the URL.
  • Body Manipulation: The attacker modifies hidden form fields or API request bodies.
  • Guessing IDs: IDs are often sequential numbers (1, 2, 3...), making it easy for an attacker to guess valid IDs of other objects.
